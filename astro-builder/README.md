# astro-builder

A Claude Code plugin for building Astro 6 static content sites — opinionated, minimal, and correct.

Enforces the [Astro 6 documentation](https://docs.astro.build/llms-small.txt) and [MDN Web API](https://developer.mozilla.org/en-US/) as its sole references. Designed for markdown-based content sites: blogs, education platforms, documentation, news sites.

Part of the [content-stack](https://github.com/pcamarajr/content-stack) marketplace.

---

## Install

```shell
/plugin install astro-builder@content-stack
```

---

## Quick start

Run `/astro-builder:init` in any Astro project (new or existing):

```shell
/astro-builder:init
```

The init skill will:
1. Scan your repo for existing assets (package.json, astro.config.ts, CSS, i18n files)
2. Interview you one question at a time (purpose, locales, content types, design system)
3. Generate `CLAUDE.md` + `.astro-builder/` configuration folder
4. Optionally scaffold missing project files and run the first build

---

## Skills

| Skill | Description |
|-------|-------------|
| `/astro-builder:init` | Initialize or re-configure a project |
| `/astro-builder:new-page` | Scaffold a page + page-view pair for all locales |
| `/astro-builder:new-content-type` | Add a new content collection with schema, utilities, and example content |
| `/astro-builder:translate` | Localize a content entry or i18n JSON to another locale |
| `/astro-builder:audit` | Audit for anti-patterns, missing translations, schema errors, and build failures |

### Examples

```shell
# Add an "about" page to all locales
/astro-builder:new-page about

# Add a nested page
/astro-builder:new-page blog/archive

# Add a dynamic route
/astro-builder:new-page tags/[tag]

# Add a new content type
/astro-builder:new-content-type tutorials

# Translate an article to Italian
/astro-builder:translate src/content/articles/en/hello-world.md it

# Translate the i18n file to Italian
/astro-builder:translate src/i18n/en.json it

# Audit the full project
/astro-builder:audit
```

## Agents

Two agents work behind the scenes:

**`astro-architect`** — Planning agent. Reads project context, designs solutions, presents implementation plans before execution. Use when you want to understand what will happen before it happens.

**`astro-builder`** — Build agent. Fully autonomous. Writes files, runs builds, fixes errors, commits. Operates without interruption until all quality gates pass.

## What gets generated by `/init`

```
your-project/
├── CLAUDE.md                          ← Project rules for Claude Code
└── .astro-builder/
    ├── style-guide.md                 ← Voice, tone, writing conventions
    ├── content-schema.md              ← Content types, fields, relationships
    ├── design-system.md               ← Colors, typography, component patterns
    └── anti-patterns.md               ← What to avoid in this project
```

Claude reads these files at the start of every session to maintain consistency across your entire project.

## What this plugin enforces

### Astro 6 patterns

- `src/content.config.ts` with `glob()` loaders (never `src/content/config.ts`)
- `astro.config.ts` in TypeScript (never `.mjs`)
- Page-views pattern: thin pages → page-views → layouts
- i18n via `Astro.currentLocale` (never URL parsing)
- `prefixDefaultLocale: true` on all routes
- `translationKey` to link content across locales
- Flexible `tags: string[]` (never fixed category enums)

### Tooling

- `pnpm` only (never npm or yarn)
- Biome for lint + format (never ESLint + Prettier)
- Official Astro integrations only (`@astrojs/sitemap`, `@astrojs/rss`, `@astrojs/mdx`)

### Quality gates (enforced on every task)

```bash
pnpm build      # Zero build errors
tsc --noEmit    # Zero TypeScript errors
biome check .   # Zero lint/format errors
```
